# 014.1 Terraform AWS EC2 SSH Key + Public IP Demo — Updated Version

**By:** Saravanan Sundaramoorthy
**Environment:** Ubuntu Linux (GCE VM)
**Time:** ~25 minutes

## Topic

This demo shows how to use **Terraform** to:

1. Install and configure AWS CLI
2. Generate an SSH key pair locally
3. Register the public SSH key in AWS using Terraform
4. Create a security group that allows SSH on port `22`
5. Create a single EC2 instance using `resource "aws_instance"`
6. Automatically fetch the latest Ubuntu AMI using Terraform
7. Output the public IP address
8. Manually SSH into the EC2 instance using the public IP
9. Destroy the infrastructure after testing

This updated version avoids hardcoding AMI IDs like:

```hcl
ami_id = "ami-00403f401ee6a4b98"
```

Instead, Terraform finds the latest Ubuntu 22.04 AMI for your selected AWS region using:

```hcl
data "aws_ami" "ubuntu" {}
```

This prevents errors like:

```text
InvalidAMIID.NotFound: The image id '[ami-xxxx]' does not exist
```

## 4. Generate SSH Key Pair Locally

Create an SSH key pair for the EC2 instance:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/terraform-aws-demo -C "terraform-aws-demo" -N ""
```

This creates two files:

```text
~/.ssh/terraform-aws-demo      # private key
~/.ssh/terraform-aws-demo.pub  # public key
```

Check files:

```bash
ls -l ~/.ssh/terraform-aws-demo*
```

Fix private key permissions:

```bash
chmod 600 ~/.ssh/terraform-aws-demo
```

View public key:

```bash
cat ~/.ssh/terraform-aws-demo.pub
```

## 8. main.tf

This is the main Terraform file.

Important change: this demo does **not** use a hardcoded `ami_id` variable.

Terraform finds the latest Ubuntu 22.04 AMI automatically for the selected region.

```bash
cat > main.tf <<'EOF_TF'
# Fetch latest Ubuntu 22.04 LTS AMI from Canonical
# Canonical owner ID: 099720109477

data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"]

  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*"]
  }

  filter {
    name   = "virtualization-type"
    values = ["hvm"]
  }

  filter {
    name   = "state"
    values = ["available"]
  }
}

# Register local public SSH key as an AWS EC2 Key Pair
resource "aws_key_pair" "demo" {
  key_name   = var.key_name
  public_key = file(pathexpand(var.public_key_path))
}

# Security group to allow SSH access
resource "aws_security_group" "ssh" {
  name        = "terraform-demo-ssh-sg"
  description = "Allow SSH access to EC2 instance"

  ingress {
    description = "SSH from allowed CIDR"
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = [var.allowed_ssh_cidr]
  }

  egress {
    description = "Allow all outbound traffic"
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Name = "terraform-demo-ssh-sg"
  }
}

# Create a single EC2 instance
resource "aws_instance" "demo" {
  ami                         = data.aws_ami.ubuntu.id
  instance_type               = var.instance_type
  key_name                    = aws_key_pair.demo.key_name
  vpc_security_group_ids      = [aws_security_group.ssh.id]
  associate_public_ip_address = true

  tags = {
    Name = var.instance_name
  }
}
EOF_TF
```

## 11. Safer SSH CIDR Option

Instead of allowing SSH from anywhere:

```hcl
allowed_ssh_cidr = "0.0.0.0/0"
```

Use your current public IP:

```bash
MY_IP=$(curl -s ifconfig.me)
echo $MY_IP
```

Then update `terraform.tfvars`:

```bash
cat > terraform.tfvars <<EOF_TF
aws_region    = "us-east-1"
instance_type = "t3.micro"
key_name      = "terraform-aws-demo-key"
allowed_ssh_cidr = "${MY_IP}/32"
instance_name = "terraform-demo-ec2"
EOF_TF
```

This allows SSH only from your current public IP.


Then initialize:

```bash
terraform init
```

Expected result:

```text
Terraform has been successfully initialized!
```

> **Note:** If you see a version constraint error like `locked provider ... does not match configured version constraint`, run:
>
> ```bash
> terraform init -upgrade
> ```

## 19. Common Error: AMI Not Found

Error:

```text
InvalidAMIID.NotFound: The image id '[ami-xxxx]' does not exist
```

Possible causes:

1. The AMI ID is from a different AWS region
2. The AMI ID is wrongly wrapped in brackets
3. `main.tf` has `ami = [var.ami_id]` instead of `ami = var.ami_id`
4. `terraform.tfvars` has a wrong AMI value

This updated demo avoids the issue by not using `ami_id` at all.

Correct approach:

```hcl
ami = data.aws_ami.ubuntu.id
```

Not:

```hcl
ami = [var.ami_id]
```

Not:

```hcl
ami = "[${var.ami_id}]"
```

---

## 20. Common Error: Instance Type Not Free Tier Eligible

Error:

```text
InvalidParameterCombination: The specified instance type is not eligible for Free Tier
```

Fix:

```hcl
instance_type = "t2.micro"
```

Or check Free Tier eligible instance types:

```bash
aws ec2 describe-instance-types \
  --region us-east-1 \
  --filters Name=free-tier-eligible,Values=true \
  --query "InstanceTypes[*].InstanceType" \
  --output table
```

Then update `terraform.tfvars`:

```hcl
instance_type = "t2.micro"
```

---

## 21. Common Error: SSH Timeout

If SSH does not connect:

```bash
ssh -i ~/.ssh/terraform-aws-demo ubuntu@PUBLIC_IP
```

Check these:

1. EC2 instance is running
2. Security group allows inbound TCP port `22`
3. `allowed_ssh_cidr` contains your current public IP
4. You are using the correct private key
5. You are using the correct username for Ubuntu AMI: `ubuntu`

Check your public IP:

```bash
curl ifconfig.me
```

If your public IP changed, update `terraform.tfvars`:

```hcl
allowed_ssh_cidr = "YOUR_PUBLIC_IP/32"
```

Then apply:

```bash
terraform apply
```

## 24. Final Concept Summary

| Concept | Meaning |
|---|---|
| `aws_key_pair` | Uploads your local public key to AWS |
| `aws_security_group` | Controls inbound and outbound traffic |
| `aws_instance` | Creates the EC2 virtual machine |
| `data "aws_ami"` | Looks up the latest Ubuntu AMI dynamically |
| `public_ip` | Public IP address used to SSH into EC2 |
| `terraform output` | Prints useful values after apply |
| `terraform destroy` | Deletes created resources |
